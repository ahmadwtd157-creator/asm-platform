cd asm-platform

# 2. شغّل النظام
docker compose up

# 3. اعمل على الكود
vim backend/app/...

docker compose up --build -d
# 5.⏹️ إيقاف المشروع

الاكتشاف (Discovery)

ASM يبحث عن:

Domains

Subdomains

IP ranges

Open ports

Web services

APIs


الجرد (Asset Inventory)

بعد الاكتشاف:

كل شيء يُخزَّن

كل أصل له:

نوع

موقع

حالة

تاريخ الاكتشاف



التحليل (Analysis)

ASM يسأل:

هل هذا الأصل مكشوف؟

هل يعمل عليه:

HTTP؟

SSH؟

Database؟

هل يستخدم:

نسخة قديمة؟

إعدادات ضعيفة؟





تقييم المخاطر (Risk)

ليس كل أصل خطر.

ASM يقيّم:

هل الأصل:

إنتاجي؟

تجريبي؟

هل عليه:

Auth؟

TLS؟

هل مكشوف للعالم؟






التنبيه والمتابعة

إذا ظهر أصل جديد:

لم يكن معروفًا

أو تم فتح Port فجأة








 Input Stage — إدخال النطاق
Domains / IPs
 المستخدم أو النظام يعطي:
    • domain رئيسي (example.com)
    • أو IP Range
Discovery Stage — الاكتشاف
DNS Scan
Subdomain Enumeration
Port Scanning
Service Fingerprinting
 هنا يعمل النظام كـ Scanner ذكي
    • يبحث
    • يجمع
    • لا يحكم
 الناتج:
Raw Assets (بيانات خام)
 Normalization & Processing — المعالجة
Normalize
Remove Duplicates
Categorize Assets
 هنا يبدأ الذكاء:
    • إزالة التكرار
    • توحيد الصيغة
    • تصنيف الأصل

Asset Inventory — التخزين
Central Asset Database
يتم حفظ:
    • كل أصل
    • حالته
    • تاريخ ظهوره

Change Detection — المقارنة
Previous State vs Current State
 النظام يسأل:
    • هل هذا أصل جديد؟
    • هل اختفى أصل؟
    • هل تغير شيء؟

 Risk Analysis — التحليل
Exposure Level
Criticality
Context
 النظام يقرر:
    • هذا خطر عالي
    • هذا عادي
    • هذا حساس
 ليس كل شيء خطر
 Alerting — التنبيه
Dashboards
Notifications
Reports
 عند وجود خطر:
    • تنبيه
    • تسجيل
    • عرض للمستخدم
Frontend — العرض
Dashboard
Asset View
Risk Timeline
 المستخدم يرى:
    • الوضع الحالي
    • التغيرات
    • المخاطر
